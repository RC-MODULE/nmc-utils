STDIO для nmc
-------------

STDIO является опциональным дополнением для libeasynmc для упрощения разработки приложений. 
Ввод/вывод со стороны linux осуществляется через устройство /dev/nmcXio, где X номер ядра. 
Устройство ввода/вывода поддерживает неблокируюий режим ввода/вывода, т.е. дескриптор 
можно передавать POLL/EPOLL. 
В случае остановки ядра NMI прерыванием POLL/EPOLL вернут HUP. По этому событию можно либо
выйти, либо игнорировать его. 

Для реализации stdio используется два кольцевых буфера в соответствующих секциях nmc-программы, 
размер которых определяется при сборке NMC проекта. 
Так как Neuromatrix DSP не имеет байтовой адресации sizeof(char)==4, то в драйвере реализовано
переформатирование ввода/вывода. По умолчанию оно всегда включено.


Для чего НЕ следует использовать stdio.
----------------------------------------

При передачи данных в stdin данные копируются два раза: 
Сначала host-системой из пространства пользователя в память nmc, с опциональным 
переформатированием, потом из кольцевого буфера копируются для дальнейшего разбора со 
стороны nmc.
Накладные расходы при копировании больших объемов данных в таком случае легко могут оказаться
больше выигрыша в производительности, который даст исполнение алгоритма на DSP.
